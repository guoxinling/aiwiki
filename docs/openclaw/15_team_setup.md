## TL;DR
通过创建 TEAM-HUB.md 和配置多个子 Agent，搭建属于你的 AI 专家团队。

## 创建团队档案

### 步骤 1: 创建 TEAM-HUB.md

```bash
cd ~/.openclaw/workspace
touch TEAM-HUB.md
```

### 步骤 2: 定义团队成员

```markdown
# 🤖 AI 团队系统

用户已经建立了一支AI专家团队，使用 `sessions_spawn` 创建子代理来模拟不同角色。

## 团队成员档案 (TEAM-HUB.md)

当用户提及团队成员或需要使用特定专家时：
1. 读取 `TEAM-HUB.md` 获取该角色的详细设定
2. 使用 `sessions_spawn` 创建一个子代理，并传入该角色的 system prompt
3. 让子代理承担该角色完成任务

## 使用方法

### 单专家模式

用户直接召唤特定专家：

```
用户: @bella 今天练习什么口语话题？
→ 创建 Bella 子代理
→ Bella: "今天我们来聊聊旅行..."
```

### 多专家协作模式

智囊大师拆解任务，调度多专家：

```
用户: @master 帮我开发一个待办事项 App
→ 智囊大师分析需求
→ 召唤 PM Mark 出需求
→ 召唤 Dev 出技术方案
→ 召唤 Designer 出UI
→ 整合完整方案汇报
```

## 使用团队成员

### 方式 1: 直接召唤

```
用户: @bella 今天练习英语口语
→ 系统自动创建 Bella 子代理
→ Bella 用英语回复
```

### 方式 2: 智囊大师协调

```
用户: @master 帮我做个网站
→ 智囊大师分析需求
→ 召唤 PM → 出需求文档
→ 召唤 Dev → 出技术方案
→ 整合后汇报给用户
```

### 方式 3: 程序化调用

```javascript
// 创建特定角色的子代理
const bella = await sessions_spawn({
  runtime: "subagent",
  task: "你是Bella，一位语言老师。教用户今天的口语话题。",
  mode: "run"
});
```

## 最佳实践

### 1. 角色互补
确保团队成员的能力互补，覆盖主要场景

### 2. 提示词清晰
每个角色的系统提示词要详细、具体

### 3. 保持简洁
不要创建过多的 Agent，3-7 个核心角色即可

### 4. 定期更新
根据实际使用情况，调整角色定义
